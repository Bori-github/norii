# 다이어그램 예시 (Mermaid)

` ```mermaid ` 펜스 안에 다이어그램을 쓰면 프리뷰에 그려진다. 소스는 그대로 텍스트로 남으므로 다른 에디터에서도 코드 블록으로 열린다.

## 플로차트

```mermaid
flowchart LR
    A[소스 텍스트] --> B[markdown-it]
    B --> C[DOMPurify sanitize]
    C --> D[프리뷰 DOM]
    D --> E{mermaid 펜스인가?}
    E -->|예| F[클라이언트에서 SVG 렌더]
    E -->|아니오| G[그대로 표시]
```

## 시퀀스 다이어그램

```mermaid
sequenceDiagram
    participant 사용자
    participant 웹뷰
    participant Rust
    사용자->>웹뷰: 파일 열기 (⌘O)
    웹뷰->>Rust: invoke("read_file")
    Rust-->>웹뷰: 본문 + 인코딩 정보
    웹뷰->>웹뷰: 에디터에 적재 · 프리뷰 렌더
    웹뷰-->>사용자: 편집 가능한 문서
```

## 상태 다이어그램

```mermaid
stateDiagram-v2
    [*] --> 저장됨
    저장됨 --> 편집중: 타이핑
    편집중 --> 저장대기: 멈춤(디바운스)
    저장대기 --> 저장됨: 자동 저장 성공
    저장대기 --> 충돌: 외부에서 파일이 바뀜
    충돌 --> 저장됨: 사용자가 선택
```

## 넓은 다이어그램 (가로 스크롤 확인용)

```mermaid
flowchart LR
    A1[하나] --> A2[둘] --> A3[셋] --> A4[넷] --> A5[다섯]
    A5 --> A6[여섯] --> A7[일곱] --> A8[여덟] --> A9[아홉] --> A10[열]
```

## 같은 다이어그램을 두 번

내용이 같으면 다시 그리지 않고 캐시에서 재사용한다.

```mermaid
flowchart LR
    A[하나] --> B[둘]
```

```mermaid
flowchart LR
    A[하나] --> B[둘]
```

## 문법이 틀린 다이어그램

앱이 깨지지 않고, 그 자리에만 오류를 알린다. 뒤에 오는 다이어그램은 정상적으로 그려진다.

```mermaid
이건 다이어그램이 아니다 {{{
```

```mermaid
flowchart LR
    앞의[앞의 오류가] --> 막지[뒤를 막지 않는다]
```

## mermaid가 아닌 코드 블록

이건 그냥 코드 블록으로 남는다.

```ts
const md = new MarkdownIt({ html: true, linkify: true });
```
