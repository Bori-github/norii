# 0007. 유리는 색이 아니라 불투명도다

- **상태**: 확정
- **날짜**: 2026-07-13

## 맥락

크롬(탭바·상태바)은 유리다 — 창 뒤 바탕화면이 흐려져 비친다(→ [0002](0002-glass-is-made-by-os.md)). 그 위에 우리는 틴트를 한 겹 얹는다. 얹지 않으면 글자가 바탕화면 위에 그대로 떠서 읽히지 않기 때문이다.

문제는 **그 틴트가 무엇이냐**였다. 팔레트를 세이지로 확정하면서(→ [0006](0006-sage-palette.md)) 틴트에도 세이지 색기를 넣었다 — `rgba(245, 249, 243, …)` / `rgba(25, 27, 24, …)`. 브랜드가 유리에까지 배어들게 하려는 의도였다.

## 결정

**틴트에서 색을 뺀다. 순백(라이트) · 순흑(다크)에 알파만 얹는다.**

```text
라이트   rgba(255, 255, 255, 0.55)
다크     rgba(0, 0, 0, 0.62)
```

유리에 색을 섞으면 **사용자의 바탕화면이 물든다.** 유리의 값어치는 "뒤에 있는 것을 그대로 보여주되 가라앉히는 것"인데, 색을 섞는 순간 그것은 유리가 아니라 색유리가 된다 — 뒤가 무엇이든 우리 색으로 덮인다. 브랜드는 액센트(→ [0005](0005-accent-is-never-text.md))와 구문 마크가 이미 지고 있고, 유리까지 브랜드를 질 필요가 없다.

**알파는 설정이 조절한다.** 값은 CSS 변수 `--norii-glass-opacity`로 두고, 토큰에는 기본값만 박는다. 설정 화면은 이 변수를 덮어써 유리를 맑게/짙게 만든다.

## 결과

- **좋은 점** — 뒤에 무엇을 두든 그 색 그대로 가라앉는다. 유리가 유리로 보인다.
- **좋은 점 2** — 알파 하나가 유일한 조절 손잡이가 된다. 설정에 노출할 것이 명확하다.
- **경계 — 알파는 아래로만 위험하다.** 기본값은 [대비 게이트](../design-system.md#대비-게이트)가 정한 하한이다: 임의의 바탕화면(순백~순흑) 위에서 크롬 글자가 AA(4.5:1)를 넘겨야 한다. 사용자가 값을 **올리면** 유리는 짙어지고 대비는 좋아지기만 하므로 안전하다. **내리는 쪽만** 위험하고, 그래서 게이트는 기본값을 검사한다.
- **따라오는 일** — 설정 화면이 이 변수의 하한을 UI에서도 강제해야 한다. 슬라이더의 최솟값이 게이트가 정한 하한과 같아야 한다(→ 열린 결정).
