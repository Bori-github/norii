# 0003. macOS 밖에서는 불투명으로 물러선다

- **상태**: 확정
- **날짜**: 2026-07-12

## 맥락

[0002](0002-glass-is-made-by-os.md)의 유리는 macOS 네이티브 재질이다. Windows는 다른 효과(Mica/Acrylic)를 갖고, Linux는 아무것도 갖지 않는다. Tauri는 플랫폼에 맞지 않는 재질을 **경고 없이 조용히 무시한다.**

## 고려한 대안

- **플랫폼별로 유리를 흉내 낸다** — Windows는 Mica, Linux는 CSS로 근사. 기각: Windows의 Mica/Acrylic은 리사이즈·드래그 시 성능 저하가 상류 문서에 명시돼 있고, Linux는 근사할 방법 자체가 없다. 무엇보다 **CSS가 플랫폼별로 갈라지기 시작한다.**
- **불투명 폴백** — 채택.

## 결정

**창 효과가 없으면 앱이 자기 캔버스 배경을 불투명하게 깔고, 크롬은 그 위에 그대로 얹힌다.** 결과적으로 비-macOS에서는 [0002](0002-glass-is-made-by-os.md)가 기각했던 "인앱 글라스"가 된다.

**플랫폼에 따라 갈라지는 CSS는 캔버스 배경 하나로 제한한다.** 이것이 지킬 수 있는 약속이다.

## 결과

- **좋은 점** — 코드가 한 갈래로 유지된다. 컴포넌트는 플랫폼을 알지 못한다.
- **나쁜 점 — 크롬 계층이 옅어진다.** "룩이 한 단계 조용해지는" 정도가 아니다. 크롬 틴트의 알파는 "임의의 바탕화면 위에서도 글자가 읽히도록" 정해지는데(→ [0004](0004-lime-is-the-only-accent.md)), 그 알파를 불투명 캔버스 위에 얹으면 크롬과 캔버스의 대비가 약해진다. **macOS 밖에서는 크롬의 경계선(border)이 계층을 대신 표시한다.**

### 되돌리기 비용 (정직한 판)

"플래그 두 개만 끄면 된다"는 말은 사실이 아니다.

```text
되돌리면 사라지는 것   tauri.conf.json 창 설정 · 플랫폼 캔버스 분기 · 창 표면 계약 테스트
되돌려도 남는 것       명시적 종이 배경 · 상태 배경 토큰(bg.hover) · 크롬 틴트 알파의 재조정
되돌릴 수 없는 것      App Store 배포 경로 (→ 0002)
```
