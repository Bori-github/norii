# 디자인 결정 기록 (ADR)

norii의 **확정된 디자인 결정**을 한 건에 한 파일로 기록한다. "왜 이렇게 정했는가"와 "무엇을 기각했는가"가 여기 산다.

## 세 문서의 경계

같은 사실을 두 곳에 적지 않는다.

```text
/DESIGN.md          지금의 규칙 — 큰 그림 · 불변식 · 표면 표 · 원칙 요약 · 이 목록의 인덱스
decisions/NNNN-*.md 그 규칙이 나온 이유 — 맥락 · 고려한 대안 · 기각 근거 · 결과
design-system.md    Panda로 어떻게 구현하는가 (계약)
window-chrome.md    창 설정의 실제 값 (계약)
```

DESIGN.md를 읽으면 **무엇을 지켜야 하는지** 알 수 있고, ADR을 읽으면 **왜 그것을 지켜야 하는지** 알 수 있다.

## 규칙

- **확정된 것만 기록한다.** 미결정은 [실제 구현 계획](../../implementation-plan.md#열린-결정-open-decisions)의 열린 결정이 단일 출처다. 두 곳에서 상태를 추적하지 않는다.
- **번호는 재사용하지 않는다.** 순차 증가만 한다.
- **삭제하지 않는다.** 결정이 뒤집히면 파일을 지우는 대신 상태를 `대체됨 (→ NNNN)`으로 바꾸고 새 ADR을 쓴다. 틀린 결정도 기록으로 남아야 같은 논의를 반복하지 않는다.
- 새 ADR은 [템플릿](0000-template.md)에서 복사해 아래 목록에 등재하고, 루트 `DESIGN.md`의 결정 목록에도 한 줄 추가한다.
- **링크는 루트 → 상세 한 방향이다.** 상세 문서는 루트를 링크하지 않는다 — 필요한 사실은 자기가 소유하거나 형제 상세 문서를 가리킨다.
