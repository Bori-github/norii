# 0005. 액센트는 테마 공통 단일 값이고, 글자로 쓰지 않는다

- **상태**: 확정 ([0004](0004-lime-is-the-only-accent.md)의 "테마가 단계를 가른다"를 대체)
- **날짜**: 2026-07-12

## 맥락

[0004](0004-lime-is-the-only-accent.md)는 액센트를 라임 하나로 정했지만, **테마가 단계를 가르게** 했다 — 다크는 형광 라임(`lime-50`), 라이트는 올리브(`lime-300`). 코드에서는 `accent` 토큰 하나만 참조하므로 문제가 없었으나, **사용자 눈에는 브랜드 색이 테마마다 다른 색으로 보인다.**

"같은 색을 두 테마에 쓸 수 있는가"를 계산했더니 답이 갈렸다.

```text
글자로 쓸 때 (AA 4.5:1)
  흰 종이 위에서 통과하려면      휘도 ≤ 0.183
  어두운 종이(#111113) 위에서는  휘도 ≥ 0.201
  → 두 조건이 모순. 라임뿐 아니라 **어떤 색도** 존재하지 않는다.

표시로 쓸 때 (비텍스트 3:1)
  휘도 0.117 ~ 0.300 이면 양쪽 통과
  → 라임 스케일에서 이 창에 드는 단계는 lime-300(휘도 0.180) **하나뿐**이다.
     흰 종이 4.56:1 · 어두운 종이 4.14:1
```

## 고려한 대안

- **다크 종이를 순흑(#000)으로 민다** — 그러면 글자 기준까지 통과한다(4.56 / 4.61). 기각: 통과 구간이 휘도 0.175~0.183으로 머리카락처럼 좁아 **여유가 사실상 0**이고, 나중에 다크 종이를 조금만 밝혀도 즉시 깨진다. 순흑 종이 자체도 장시간 읽기에 대비가 과하다.
- **테마별 단계 유지(0004 그대로)** — 기각: 브랜드 색이 테마마다 달라 보이는 문제가 그대로다.
- **액센트를 글자로 쓰지 않는다** — 채택.

## 결정

**액센트는 `lime-300` 하나이고, 테마에 따라 갈리지 않는다.**

**액센트를 글자에 쓰지 않는다.** 액센트는 **살아 있는 것의 표시**에만 쓴다 — 커서 · 활성 탭의 dirty ● · 포커스 링 · 강조 테두리. 전부 비텍스트라 3:1 기준이 적용된다.

강조가 필요한 버튼(배너 액션·다이얼로그 확인)은 **글자색이 아니라 테두리와 굵기**로 강조한다.

## 결과

- **좋은 점** — 브랜드 색이 테마와 무관하게 한 색이다. 그리고 0004가 세운 원칙("액센트는 살아 있는 것에만")에 **오히려 더 충실해진다** — 액센트가 글자에서 빠지면서 쓰이는 자리가 더 줄었다.
- **좋은 점** — 다크 종이를 순흑으로 밀지 않아도 되고, 대비 여유가 남는다.
- **나쁜 점 — 형광이 사라진다.** 다크에서 빛나던 `#CAFB41`은 앱 UI에서 쓰이지 않는다. 액센트는 항상 차분한 올리브다. "형광으로 기억되는 화면"은 포기한 것이다.
- **나쁜 점** — 강조 버튼의 시각적 무게가 약해진다. 글자색이라는 가장 강한 신호를 테두리·굵기로 대체했다.
- **따라오는 일** — 게이트가 이 결정을 강제한다. 액센트가 **어느 한 테마에서 글자 기준(AA)을 넘어버리면 테스트가 실패한다** — 그러면 금지의 근거가 사라지므로 규칙과 팔레트 중 하나를 고쳐야 한다는 신호다(→ [대비 게이트](../design-system.md#대비-게이트)).
