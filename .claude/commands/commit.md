관련 변경을 컨벤션에 맞게 커밋한다. 규칙의 단일 출처는 `.claude/rules/commit-convention.md`이며, 이 커맨드는 그것을 실행할 뿐 규칙을 다시 정의하지 않는다.

## 절차

1. **변경 파악** — `git status --short`와 `git diff`(스테이징/미스테이징)로 무엇이 바뀌었는지 본다. 서로 다른 관심사가 섞였으면 나눠 커밋할지 사용자에게 묻는다(한 커밋 = 한 논리 단위).
2. **스테이징** — 이 커밋에 넣을 파일만 `git add <경로>`로 담는다. 의도치 않은 파일을 `git add -A`로 쓸어 담지 않는다.
3. **문서 동기화** — 동작을 바꾸는 변경이면 해당 계약 문서(→ [작업 규칙](../rules/project-rules.md))가 **같은 커밋에** 있는지 확인한다. 없으면 먼저 갱신한다. 설계와 달라졌으면 임의 진행하지 말고 보고한다.
4. **메시지 작성** — 형식·타입·스코프·요약 규칙·본문 길이·`Co-Authored-By` 트레일러는 [커밋 컨벤션](../rules/commit-convention.md)을 그대로 따른다(여기 다시 나열하지 않는다).
5. **커밋** — 메시지를 보여주고 `git commit`으로 커밋한다. lefthook `commit-msg` 훅(commitlint)이 검증하며, 거부되면 메시지를 고쳐 다시 시도한다.

## 규칙

- 깨진 테스트·편집 중간 상태를 커밋하지 않는다.
- **push는 하지 않는다** — 이 커맨드는 로컬 커밋까지다. push·PR은 별도 지시가 있을 때만.
